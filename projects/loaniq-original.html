03 — LangGraph Multi-Agent Pipeline
AGENT 01
🛂
IntakeAgent — Front Door Examiner
intake_agent.py · GPT-4o-mini · START node
▼ 4 steps
✅
RULES Field-level validation
credit_score ∈ [300,850] · annual_income > 0 · property_value > 0 · loan_amount > 0 · loan_type ∈ VALID_LOAN_TYPES · occupancy ∈ VALID_OCCUPANCIES · property_type ∈ VALID_PROPERTY_TYPES · term ∈ [10,15,20,25,30]. Returns errors[] if any fail.
🧮
Derived field computation
monthly_gross_income = annual_income / 12. Stored once in state — all downstream agents read it without recomputing. Consistent source of truth.
🤖
GPT-4o-mini Semantic consistency check
Prompt: "Review borrower profile for semantic inconsistencies." temp=0 + response_format=json_object. Returns: data_quality_score · income_plausibility · employment_assessment · flag_anomalies[]. Graceful fallback on LLM error — validation continues without it.
🚦
CONDITIONAL route_after_intake()
LangGraph add_conditional_edges: if len(state.errors) > 0 with critical flags → route to "audit" (fast fail, skip all agents). Otherwise → "ratio_calculation". Critical = missing required field, invalid loan_type, credit_score out of range.
AGENT 02
📐
RatioCalculationAgent — The Actuary
ratio_agent.py · Pure Math + GPT-4o narrative · No hallucination possible
▼ 5 steps
🏠
MATH LTV / CLTV / HLTV
LTV = loan_amount / property_value. CLTV = (loan + existing_mortgage) / value. HLTV = max HELOC exposure / value. Decimal(10, ROUND_HALF_UP) for precision. All stored as ratio (0.8444) AND percentage (84.44) — downstream agents use both forms.
💵
MATH Monthly payment + rate estimation
M = P[r(1+r)^n] / [(1+r)^n−1]. Rate from base (6.5%) adjusted by: credit_score buckets (LLPA) · loan_type (FHA +0.25%, VA −0.25%, JUMBO +0.5%) · term (15yr −0.75%). PMI added if LTV > 80% CONVENTIONAL.
⚖️
MATH Front-end + Back-end DTI
front_end_dti = monthly_payment / gross_income. back_end_dti = (payment + all_monthly_debts) / gross_income. QM threshold = 43%. Both as decimal and pct. VA/USDA use residual income model instead.
🚩
Eligibility pre-check flags
ltv_within_standard_limit (≤80%) · pmi_required (LTV>80% CONV) · dti_within_qm_limit (≤43%) · credit_score_meets_minimum (CONV:620, FHA:580, VA:620, JUMBO:700, USDA:640). Flags stored as booleans in state.
🤖
GPT-4o Ratio narrative for UnderwritingAgent
LLM given all computed numbers. Generates 3–4 sentence ratio_narrative: strengths, concerns, compensating factors. Stored in state — UnderwritingAgent includes it verbatim in its own context. Cross-agent knowledge transfer.

04 — Advanced RAG Pipeline (called by PolicyRetrievalAgent)
🔮
8-STAGE ADVANCED RAG PIPELINE
app/services/rag/ · query_rewriter → hybrid_retriever → reranker → context_builder → generator → output_validator
▼ expand
STAGE 01
QueryRewriter  GPT-4o-mini
Rewrites raw agent query for retrieval clarity. Expands into 3 sub-queries (multi-query expansion). Each sub-query hits retrieval independently → broader policy coverage. LangSmith child span created.
STAGE 02
HybridRetriever  BM25 pgvector
BM25 keyword search (rank-bm25, in-memory over loaded chunk corpus). Dense vector search (text-embedding-3-small, 1536-dim, cosine similarity via pgvector IVFFlat index). Reciprocal Rank Fusion (RRF) merges both ranked lists → 65 candidates. HYBRID_BM25_WEIGHT = 0.3 (configurable).
STAGE 03
MMRFilter  numpy
Maximum Marginal Relevance. λ=0.5 (MMR_LAMBDA, configurable). Balances relevance vs diversity. Prevents returning 5 nearly-identical chunks about the same policy section. Cosine similarity matrix computed in-memory with numpy. Reduces to TOP_K_CANDIDATES (15).
STAGE 04
CrossEncoderReranker  LOCAL · FREE CPU
cross-encoder/ms-marco-MiniLM-L-6-v2 via sentence-transformers. Reads BOTH query + chunk together in full attention — far more accurate than cosine similarity alone. Typical scores: −4 to +8. Top 5 chunks selected after reranking. No API cost. Runs on CPU (slow but free).
STAGE 05
ContextBuilder
Deduplicates near-identical chunks (token-level Jaccard overlap threshold). Token-budget-aware selection (max 3000 tokens). Each chunk formatted: "[SOURCE N: {document_title}, section: {section_title}]\n{content}". Metadata preserved for citation validation downstream.
STAGE 06
RAGGenerator  GPT-4o
System prompt: "You are a mortgage policy expert. Answer ONLY from provided sources. Cite each fact as [SOURCE N]." Returns JSON: answer · citations[{source_index, text}] · confidence (0–1) · is_answerable · reasoning. temp=0 for determinism.
STAGE 07
OutputValidator  ECOA Guard
Citation integrity: all [SOURCE N] refs must match a real chunk index. ECOA word-boundary regex: re.search(r'\b' + term + r'\b', answer, re.IGNORECASE) — checks for protected class mentions (race, religion, national origin, sex, marital status, age, familial status). Confidence threshold check. is_valid + failed_checks[] returned.
RESULT
RAGResult dataclass
answer · citations · confidence · is_answerable · is_valid · failed_checks · retrieved_chunks[5] · query_used · processing_stages_ms dict · langsmith_trace_url. Returned to PolicyRetrievalAgent → written to state['retrieved_policy_chunks'].
